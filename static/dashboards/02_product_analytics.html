// Python: Cohort Builder

Built a reusable cohort analysis class in pandas that any PM can run with a single function call — reduced ad-hoc retention queries from 2 hours to 3 minutes.

02 The Python — Cohort Retention Engine pandas · reusable class · used by 4 PMs
# cohort_analysis.py — reusable retention engine # Reduced ad-hoc analysis time: 2hrs → 3 mins import pandas as pd import numpy as np class CohortRetentionAnalyser: def __init__(self, events_df: pd.DataFrame): # expects: user_id, event_date, signup_date self.df = events_df.copy() self.df['signup_month'] = pd.to_datetime( self.df['signup_date'] ).dt.to_period('M') self.df['days_since_signup'] = ( pd.to_datetime(self.df['event_date']) - pd.to_datetime(self.df['signup_date']) ).dt.days def build_cohort_matrix( self, periods=[1,7,14,30,60,90] ) -> pd.DataFrame: # Count users active within N days of signup cohort_sizes = ( self.df.groupby('signup_month')['user_id'] .nunique() .rename('cohort_size') ) matrix = {} for p in periods: active = ( self.df[self.df['days_since_signup'] <= p] .groupby('signup_month')['user_id'] .nunique() ) matrix[f'D{p}'] = (active / cohort_sizes * 100).round(1) return pd.DataFrame(matrix) def find_dropoff_cliff(self) -> dict: # Identify the period with steepest avg retention drop m = self.build_cohort_matrix() diffs = m.diff(axis=1).mean() cliff = diffs.idxmin() return {'cliff_period': cliff, 'avg_drop_pp': round(diffs[cliff], 1)}
-- SQL: Feature activation within first 7 days -- Used to identify "power features" correlated -- with D30 retention (activation metric framework) WITH new_users AS ( SELECT user_id, signup_date FROM users WHERE signup_date >= '2024-01-01' ), feature_activations AS ( SELECT e.user_id, e.feature_name, MIN(e.event_date) AS first_used, DATE_DIFF( MIN(e.event_date), u.signup_date, DAY ) AS days_to_activate FROM events e JOIN new_users u USING(user_id) WHERE e.event_date BETWEEN u.signup_date AND u.signup_date + INTERVAL 7 DAY GROUP BY 1,2 ), retained AS ( SELECT DISTINCT user_id FROM events WHERE DATE_DIFF(event_date, (SELECT signup_date FROM new_users WHERE user_id = events.user_id), DAY) BETWEEN 25 AND 35 ) SELECT fa.feature_name, COUNT(DISTINCT fa.user_id) AS activated_users, COUNT(DISTINCT r.user_id) AS retained_users, ROUND(COUNT(DISTINCT r.user_id) * 100.0 / COUNT(DISTINCT fa.user_id),1) AS retention_rate FROM feature_activations fa LEFT JOIN retained r USING(user_id) GROUP BY 1 ORDER BY retention_rate DESC
// KEY INSIGHT: "Power Feature" Discovery

Users who activate the Team Collaboration feature within 7 days retain at 71% vs 28% for non-activators. This became the primary onboarding activation metric — now tracked as a north-star input KPI.

✓ WINNER — Interactive Tutorial +24% CVR
Replaced 5-step text onboarding with interactive product tour. Stat sig at 95% (p=0.031). D7 retention: 64% vs 52% control. Rolled out to 100% of new signups.
~ INCONCLUSIVE — Checklist UI +3% CVR
Onboarding checklist vs. progress bar. Not stat sig (p=0.24). Requires 3× more traffic. Extended test for 60 more days before decision.
✗ LOSER — Forced Feature Tour −8% CVR
Mandatory 8-step modal sequence before dashboard access. Users rage-quit at step 3. Immediate rollback. Qualitative: users felt "trapped." Skip logic is mandatory.
